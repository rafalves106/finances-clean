## 🔴 Achados Críticos (bloqueiam merge)

### [SEC-001] Segredos reais versionados no backend (`appsettings.json`)

- **Arquivo**: `server/API/appsettings.json:10`
- **Categoria**: Secret commitado
- **OWASP**: A02:2021 – Cryptographic Failures / A05:2021 – Security Misconfiguration
- **Evidência (comando)**:
  - `git blame -L 10,19 API/appsettings.json` (executado em `server/`)
  - `read_file server/API/appsettings.json`
- **Evidência (conteúdo)**:

```json
"PostgresConnection": "Host=localhost;Port=5432;Database=FinanceDb;Username=falves;Password=__REDACTED_DB_PASSWORD__"
"Key": "__REDACTED_JWT_KEY__"
"AdminKey": "__REDACTED_ADMIN_KEY__"
```

- **Risco**: acesso indevido ao banco, forja de JWT válidos e criação de contas administrativas via header `X-Admin-Key`.
- **Correção sugerida**:
  1. Remover segredos de `appsettings.json` e usar variáveis de ambiente/secret manager.
  2. Manter apenas placeholders em `appsettings.Example.json`.

### [SEC-002] CORS permissivo global (`AllowAnyOrigin`) em política aplicada à API

- **Arquivo**: `server/API/Program.cs:107`
- **Categoria**: Security Misconfiguration
- **OWASP**: A05:2021 – Security Misconfiguration
- **Evidência (comando)**:
  - `grep -RInE "AllowAnyOrigin|..." --include="*.cs" server/API ...`
- **Evidência (conteúdo)**:

```csharp
policy.AllowAnyOrigin()
      .AllowAnyHeader()
      .AllowAnyMethod();
```

- **Risco**: qualquer origem pode chamar endpoints autenticados a partir de browser, ampliando superfície para abuso de token e exfiltração.
- **Correção sugerida**:

```csharp
policy.WithOrigins(allowedOrigins)
      .AllowAnyHeader()
      .AllowAnyMethod();
```

### [SEC-006] Operações financeiras multi-escrita sem transação explícita

- **Arquivo**: `server/Core/UseCases/Investimento/RealizarAporteUseCase.cs:23`
- **Categoria**: Integridade transacional
- **OWASP**: A04:2021 – Insecure Design
- **Evidência (comando)**:
  - `read_file` dos use cases e repositórios
  - `grep -RInE "BeginTransaction|Database\.BeginTransaction|SaveChanges\(" ...`
- **Evidência (conteúdo)**:
  - `RealizarAporteUseCase` grava em `_investimentoRepository.Atualizar(...)` e `_movimentacaoRepository.Adicionar(...)`

## 🟡 Achados Médios

### [SEC-009] TTL de JWT excessivo para contexto financeiro

- **Arquivo**: `server/Infrastructure/Services/TokenService.cs:27`
- **Evidência**: `var expiryDays = int.Parse(_configuration["Jwt:ExpiryDays"] ?? "7");`
- **Risco**: janela maior de abuso em caso de token comprometido.
- **Correção sugerida**: reduzir para <= 1h + refresh token seguro/rotativo.

### [SEC-010] Ausência de headers HTTP de hardening (CSP, HSTS, X-Frame-Options)

- **Arquivo**: `server/API/Program.cs` (sem configuração desses headers)
- **Evidência (comando)**: `grep -RInE "X-Frame-Options|Content-Security-Policy|Strict-Transport-Security" --include="*.cs" server` retornou vazio.
- **Risco**: superfície ampliada para clickjacking e ataques de conteúdo.

### [SEC-011] Endpoints públicos de auth não marcados explicitamente com `[AllowAnonymous]`

- **Arquivo**: `server/API/Controllers/AuthController.cs:8`
- **Evidência (comando)**: `grep -RIn "\[AllowAnonymous\]" --include="*.cs" API/Controllers` retornou vazio.
- **Risco**: ambiguidade operacional/futura regressão de política de acesso.

### [SEC-012] Validação de entrada fraca/implícita em DTOs de autenticação

- **Arquivo**: `server/Core/Application/DTOs/Auth/LoginDTO.cs:3`, `server/Core/Application/DTOs/Auth/RegistroDTO.cs:3`
- **Evidência**: DTOs sem atributos de validação (`[Required]`, tamanho mínimo, formato).
- **Risco**: payloads malformados aumentam chance de erro e bypass de expectativas de negócio.

### [SEC-013] Uso de `DateTime.Now` em fluxo financeiro

- **Arquivo**: `server/Core/UseCases/Investimento/RemoverInvestimentoUseCase.cs:28`
- **Evidência**: `DateTime.Now` ao registrar estorno.

3. Corrigir CORS para allowlist (`WithOrigins(allowedOrigins)`).

### Fase 2 — Limpeza de histórico Git (obrigatória)

#### Opção A: `git filter-repo` (recomendado)

```bash
# 1) Clone espelho
git clone --mirror <REMOTE_URL> finances-mirror.git
cd finances-mirror.git

# 2) Arquivo de substituições
cat > replacements.txt << 'EOF'
literal:__REDACTED_JWT_KEY__==>REDACTED_JWT_KEY
literal:__REDACTED_ADMIN_KEY__==>REDACTED_ADMIN_KEY
literal:Password=__REDACTED_DB_PASSWORD__==>Password=REDACTED_DB_PASSWORD
literal:__REDACTED_BOOTSTRAP_PASSWORD__==>REDACTED_SEEDED_PASSWORD
EOF

# 3) Reescrever histórico
git filter-repo --replace-text replacements.txt

# 4) Forçar push
git push --force --all
git push --force --tags
```

#### Opção B: BFG Repo-Cleaner

```bash
git clone --mirror <REMOTE_URL> finances-mirror.git
cd finances-mirror.git

cat > secrets.txt << 'EOF'
__REDACTED_JWT_KEY__
__REDACTED_ADMIN_KEY__
Password=__REDACTED_DB_PASSWORD__
__REDACTED_BOOTSTRAP_PASSWORD__
EOF

java -jar bfg.jar --replace-text secrets.txt
git reflog expire --expire=now --all
git gc --prune=now --aggressive
git push --force --all
git push --force --tags
```

### Fase 3 — Pós-limpeza (obrigatório)

1. Revogar clones antigos e pedir re-clone para todo o time.
2. Invalidar credenciais antigas no provedor (DB/infra).
3. Ativar scanner preventivo (gitleaks/trufflehog + bloqueio em CI).
