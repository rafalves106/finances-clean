## E. Achados detalhados

### 🔴 [UX-001] Simulador de juros permite taxa 0 sem feedback de erro

- **Arquivo**: client/src/components/InvestmentsView.jsx
- **Heurística violada**: #5 Prevenção de erros
- **Risco**: cálculo inválido, resultado infinito/NaN, confunde usuário
- **Sugestão**: bloquear taxa 0 e exibir mensagem clara "Taxa deve ser maior que zero"
- **Referência**: WCAG 3.3.1, NN/g Error Prevention

### 🟠 [UX-002] Exclusão destrutiva sem confirmação em alguns fluxos

- **Arquivo**: client/src/components/TransactionModal.jsx, WishListView.jsx, VehicleView.jsx
- **Heurística violada**: #3 Controle e liberdade
- **Risco**: usuário pode excluir sem querer, sem desfazer
- **Sugestão**: modal de confirmação ou toast com desfazer
- **Referência**: NN/g Confirmation Dialog

### 🟠 [UX-003] Mensagens de erro não acessíveis (sem aria-live)

- **Arquivo**: vários (alert/modal)
- **Critério WCAG**: 4.1.3 Status Messages
- **Sugestão**: usar div com `role="alert"` ou aria-live

### 🟡 [UX-004] Botões de ação só com ícone sem label acessível

- **Arquivo**: DashboardView.jsx, InvestmentsView.jsx, CategoryManagerModal.jsx
- **Critério WCAG**: 2.5.3 Label in Name
- **Sugestão**: adicionar `aria-label` nos botões de ícone

### 🟡 [UX-005] HTML root com lang="en" em vez de pt-BR

- **Arquivo**: client/index.html
- **Sugestão**: `<html lang="pt-BR">` para acessibilidade e SEO

### 🟡 [UX-006] Estados de vazio/sucesso ausentes em alguns fluxos

- **Arquivo**: InvestmentsView.jsx, VehicleView.jsx
- **Sugestão**: adicionar empty state e feedback de sucesso

### 🟢 [UX-007] Sugestão: toast para feedback de sucesso/erro

- **Arquivo**: geral
- **Sugestão**: usar componente de toast acessível

### 🟢 [UX-008] Sugestão: tokens customizados para cores/espacamento

- **Arquivo**: tailwind.config.js
- **Sugestão**: definir tokens custom para facilitar manutenção
